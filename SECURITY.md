# Security Policy

## Supported Versions

We currently provide security fixes for the latest commit on the default branch.

| Version | Supported |
| --- | --- |
| Default branch | :white_check_mark: |
| Older commits / tags | :x: |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security-sensitive reports.

Preferred: report privately via GitHub Security Advisories:
https://github.com/woodstock-tokyo/pinescription/security/advisories/new

If you cannot use GitHub Security Advisories, email: support@woodstock.co  
(please include `SECURITY:` in the subject)

When reporting, include as much as you can:

- A description of the issue and potential impact
- Steps to reproduce / proof-of-concept
- Affected versions/commits (if known)
- Any relevant logs, stack traces, or patches

## Coordinated Disclosure

We will acknowledge receipt and work with you on a reasonable timeline for a fix and public disclosure. Please keep details private until a fix is available.
